## ⚠️ ВАЖНО: как собирать ZIP

**Архив собирать ТОЛЬКО через Проводник Windows** (правой кнопкой по папке `out/` → «Сжать в ZIP-файл»). 

**НЕЛЬЗЯ** использовать:
- ❌ PowerShell `Compress-Archive` — ломает папку `404/` (создаёт рядом мусорный файл `404.html` → Beget путает файл и папку → все подстраницы 404)
- ❌ `7-Zip` со стандартными настройками — нужно явно отключать нормализацию имён
- ❌ любые архиваторы, которые «упрощают» имена файлов

**Можно** (если хочется из консоли):
- ✅ Git Bash: `cd hib && tar -a -c -f ../hib-deploy.zip out` 
- ✅ PowerShell: `tar -a -c -f hib-deploy.zip -C hib out` (встроенный `tar` в Windows 10+ работает корректно)
- ✅ 7-Zip с параметрами: `7z a -tzip -mx=5 hib-deploy.zip hib/out`

Подробнее о баге с `Compress-Archive` — в «Проблеме 0.5» внизу инструкции.

### 6. ⚠️⚠️ САМЫЙ ВАЖНЫЙ ШАГ: смени права доступа

**Это и есть причина 404 на вложенных страницах!**

При распаковке Beget выставляет права **`600`** (только владелец может читать). Beget использует разные UID для пользователя и для Apache. Apache под другим UID **не может прочитать файлы с правами 600** → 404.

**Решение — 3 способа (от простого к правильному):**

#### Способ A: одна команда в GUI (самый быстрый)

1. В файловом менеджере Beget перейди в `hib/`
2. На **пустом месте** (где нет файлов) нажми правой кнопкой → **«Атрибуты»** / **«Права»** / **«Chmod»**
3. Поставь кодовое значение **`755`**
4. Поставь галочку **«Применить рекурсивно ко всем вложенным файлам и папкам»**
5. Нажми «Применить» / «Изменить»

Это установит `755` на всё (папки и файлы). 755 на файлах — лишний бит выполнения, но Apache всё равно прочитает.

#### Способ B: правильный (файлы 644, папки 755)

В стандартной модели Unix:
- **Файлы** должны иметь права **`644`** (rw-r--r--)
- **Папки** должны иметь права **`755`** (rwxr-xr-x)

1. Выдели **все файлы** в `hib/` (Ctrl+A в корне папки)
2. Правой кнопкой → «Атрибуты» → `644` → Применить
3. Выдели **все папки** в `hib/`
4. Правой кнопкой → «Атрибуты» → `755` → Применить
5. Зайди в каждую вложенную папку (`_next/`, `training/`, `dashboard/`, `about/`, `research/`, `games/`, `404/`) и **повтори шаги 1-4** для её содержимого. В файловом менеджере Beget это можно сделать **рекурсивно** — выдели все вложенные папки/файлы сразу и примени 755/644.

#### Способ C: через SSH (самый правильный, но нужен доступ)

1. В карточке на главной Beget включи **SSH-доступ** (переключатель «SSH-доступ» — у тебя сейчас выключен)
2. Подключись по SSH: `ssh saguitv5@saguitv5.beget.tech`
3. Выполни:

```bash
cd ~/sk108projects.ru/public_html/hib
find . -type d -exec chmod 755 {} \;   # все папки = 755
find . -type f -exec chmod 644 {} \;   # все файлы = 644
```

Это правильный способ: рекурсивно все папки получают 755, все файлы — 644.

## Возможные проблемы и решения

### Проблема 0.5: ZIP-архив повреждён (Compress-Archive / 7-Zip)

**Симптомы:**
- В файловом менеджере Beget после распаковки **папки отображаются как файлы 0 байт**: `404.html` (вместо папки `404/`), `_next/3DL7X9Ugtaqe6qpRV6VT_` (0 байт), `_next/static/` (0 байт), и т.д.
- `hib/index.html` отдаётся 200, всё остальное — 404
- При ручном открытии ZIP на компе — **папки в нём видны нормально**, никакой ошибки нет

**Причина:** ZIP был собран через **PowerShell `Compress-Archive`** (или 7-Zip со стандартными настройками). `Compress-Archive` при наличии в архиве папки `404/` создаёт в корне **дополнительный файл `404.html`** (видимо, конфликт имён с HTTP-кодом 404). Beget-файловый менеджер при распаковке принимает `404.html` за папку, остальные папки становятся «пустыми файлами». nginx/Apache видят пустые 0-байтовые папки, отдают 404.

**Решение:**
1. Не пересобирай архив по сто раз — проверь текущий ZIP:
   ```bash
   unzip -l hib-deploy-2026-06-12.zip | head -10
   ```
   Если в начале списка видишь **`404.html`** (файл, не папка) рядом с `404\index.html` — архив битый.
2. Пересобери ZIP **через Проводник Windows**: открой папку `hib/`, правой кнопкой по `out/` → «Сжать в ZIP-файл».
3. Либо из консоли (Git Bash): `cd hib && tar -a -c -f ../hib-deploy.zip out`
4. Перезалей архив в Beget, удали старый `.htaccess` и все `404/`, `_next/`, `training/` (если они там остались как 0-байтовые файлы), распакуй заново.

**Почему это самая коварная проблема:** в файловом менеджере Beget **всё выглядит правильно** — структура папок на месте, иконки те же, только размер у некоторых папок `0 byt...` (видно на скриншоте у пользователя). Глаз цепляется за то, что папки есть, и ты думаешь «всё ок, дело в правах». Права тут ни при чём — архив битый.

**Эта проблема убила ~2 часа деплоя 2026-06-12**, потому что я и Сергей по очереди меняли права, искали конфиги nginx, проверяли `.htaccess` — а на самом деле надо было просто пересобрать ZIP в Проводнике.

### Проблема 0 (ГЛАВНАЯ): главная работает, а все остальные страницы и стили — 404

**Симптомы:**
- `https://sk108projects.ru/hib/` показывает страницу (но без CSS и JS — голый HTML)
- `https://sk108projects.ru/hib/training/` → 404
- `https://sk108projects.ru/hib/_next/static/css/...` → 404
- В файловом менеджере Beget все файлы на месте, **атрибуты `600`**

**Причина:** права доступа `600` (только владелец). Apache на shared-тарифах работает под другим UID и не может прочитать эти файлы.

**Решение:** см. **Шаг 6** выше.

### Проблема 1: 404 Not Found на https://sk108projects.ru/hib/

**Причина:** папка `hib` лежит не там, где Beget ищет файлы сайта.

**Решение:** проверь, что ты залил в правильную папку. На Blog-тарифе Beget корневая папка для домена называется `public_html`. То есть правильный путь:

```
/saguitv5.beget.tech/www/sk108projects.ru/public_html/hib/
```

Если папки `public_html` нет, ищи `sk108projects.ru/hib/` напрямую.

### Проблема 2: при переходе на /hib/training/ получаю 404

**Причина:** static export создаёт файлы `training/index.html`, а сервер Beget (Apache) не всегда автоматически отдаёт `index.html` по запросу `/training/`.

**Решение:** в новой сборке я уже добавил `.htaccess` с правилом `DirectoryIndex index.html`. Если не сработало — проверь, что `.htaccess` залился в корень `hib/` (он есть в новом ZIP, в предыдущем архиве его не было).

### Проблема 3: CSS не подгружается (страница без стилей)

**Причина:** папка `_next/` не залита, не распакована или имеет неправильные права.

**Решение:** проверь, что внутри `hib/_next/` есть `static/css/` с CSS-файлами. Если её нет — перезалей ZIP и распакуй заново. Если папка есть, но стили не грузятся — **проверь права** на `_next/`, `_next/static/`, `_next/static/css/`, `_next/static/chunks/` (должны быть 755 у папок и 644 у файлов).

### Проблема 4: «localStorage is not defined» в консоли

**Причина:** не должно быть в нашем проекте (мы везде проверяем `typeof window !== "undefined"`), но если вдруг появится — обнови страницу с очисткой кэша (Ctrl+F5).

### Проблема 5: тема не переключается (всё время тёмная)

**Причина:** не страшно. По умолчанию HIB = тёмная. Кликни на иконку солнца/луны в правом верхнем углу шапки.
